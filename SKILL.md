# 今日头条时事评论创作技能（jrtt-skill）

> ⚠️ **触发互斥声明**：本skill仅在用户明确提到"头条"相关词时启动。若用户只说"写文章"、"写一篇"等泛用指令，由其他写作skill处理，本skill不介入。

---

## 用户画像（始终牢记）

- **风格定位**：理性深度派——两边观点都摆，最后旗帜鲜明站一边
- **粉丝关系**：粉丝跟他是因为"帮我把复杂事情讲清楚了"
- **发布频率**：每日更新
- **三大痛点**：热点发现慢 / 标题卡死区 / 不会设计甜蜜争议点

### 第3步：标题工厂

**基于真实素材**，生成5个标题：

```

━━━━━━━━━━━━━━━━━━━━━━━━

【认知颠覆型】③ [标题] 【X字 ✅/❌】
【深度切入型】④ [标题] 【X字 ✅/❌】

📌 推荐 ③，原因：[一句话]

```

**标题铁律：**
- 字数22-30字，硬性上限30字，超过自动重写
- 每个标题自动标注字数，超30字标❌并重写
- 必含真实数字或对比，数字来自搜索结果
- 不用"震惊""炸锅"等过度词

**等待用户确认标题后进入第4步。**

### 第4步：正文生成

**所有数据、案例、观点必须来自第0.5步和第2.5步的搜索结果，禁止编造。**

结构模板（理性深度派专用）：

- **黄金开头（100-150字）**：用搜索到的真实场景或数据开场，第一句必须是画面或数字
- **现象还原（250字）**：引用搜索到的真实数据和案例，每300字设一个小转折
- **两边的声音（350字）**：直接引用素材库中两边最强论据，两边都摆得有力
- **我的判断（350字）**：旗帜鲜明站边→指出两边盲点→用搜索发现的独家角度收尾
- **升华结尾（150字）**：金句收尾 + 甜蜜争议点

**禁用词**：震惊/炸锅/燃爆/破防；近年来/随着……的发展；总之/综上所述

**数据标注规则**：正文中每个数据后面用括号标注来源，例如：
> 128万人次（今日头条2024年数据）
> 73.6%增长率（百度新闻·国家卫健委）

---

### 第5步：甜蜜争议点设计

**三种公式（正文结尾必用其一，问题要基于搜索发现的真实争议）：**

| 公式 | 模板 | 适用场景 |
|------|------|----------|
| A·开放选择题 | "你是[立场1]还是[立场2]？评论区聊聊" | 有明显两派 |
| B·经验召唤 | "你/身边有没有[相关经历]？" | 话题贴近生活 |
| C·认知挑战 | "如果我说[反直觉结论]，你同意吗？必须二选一你选哪个？" | 文章有独特判断 |

**禁止**：道德审判式 / 非此即彼绝对化 / 暗含答案的假问句

## Word文档输出

**第4步正文定稿后自动执行，保存到 `E:/VSCODE/文章/`。**

### 依赖检查（首次使用前执行一次）

```bash
node --version
cd %USERPROFILE%\.claude\skills\jrtt-skill\scripts && npm install docx
```

### 执行步骤

**Step 1：生成 JSON 内容文件** `article_input.json`

```json
{
  "title": "文章完整标题",
  "sections": [
    { "type": "body",    "text": "正文段落" },
    { "type": "bold",    "text": "加粗段落" },
    { "type": "quote",   "text": "引用框内容" },
    { "type": "heading", "emoji": "📊", "text": "章节标题" },
    { "type": "divider" },
    { "type": "spacer",  "pt": 10 },
    { "type": "slogan",  "text": "居中金句" },
    { "type": "debate",  "text": "争议互动框内容" },
    { "type": "image",   "text": "中文关键词 → https://unsplash.com/s/photos/keyword" }
  ]
}
```

| type | 效果 |
|------|------|
| `body` | 普通正文，行距1.5倍 |
| `bold` | 加粗正文，关键判断句 |
| `quote` | 蓝底左红线引用框 |
| `heading` | 红色章节标题，带emoji |
| `divider` | 灰色分隔线 |
| `spacer` | 空白间距 |
| `slogan` | 居中红色金句 |
| `debate` | 暖色争议框 |
| `image` | 绿底可点击配图链接 |

**Step 2：调用生成脚本**

```bash
node %USERPROFILE%\.claude\skills\jrtt-skill\scripts\gen_docx.js article_input.json
```

默认输出到 `E:/VSCODE/文章/[文章标题].docx`
